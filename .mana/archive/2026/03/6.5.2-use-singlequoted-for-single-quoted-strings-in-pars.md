Change parser to emit Argument::SingleQuoted for Token::SingleQuotedString.
Currently both single and double quoted strings produce Argument::Literal, so
the executor expands variables in single-quoted strings (wrong).

In src/parser/mod.rs, find every Token::SingleQuotedString handler and change
Argument::Literal(unquoted) to Argument::SingleQuoted(unquoted).
Search: rg "SingleQuotedString" src/parser/mod.rs

Key locations: parse_argument (~line 698), parse_word, parse_redirect_target,
parse_expression, parse_pipe_ask_prompt, parse_assignment_value, parse_case_pattern.

For expression/redirect/assignment contexts that need a String (not Argument),
just use the unquoted string directly — single quotes never expand.

Fixes: test_single_quotes_preserve_literal_dollar, test_single_quotes_preserve_backticks,
test_command_substitution_blocked_in_single_quotes
