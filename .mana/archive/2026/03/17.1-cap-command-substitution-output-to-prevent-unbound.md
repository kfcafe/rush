## Problem
`execute_command_substitution()` in `src/executor/expansion.rs` captures the entire stdout of a child command into a `String` with no size limit. Running `$(cat /dev/urandom | head -c 1G)` would allocate 1GB+ of RAM.

Same pattern exists in `execute_command_substitution_str()` and the two `_static` variants in the same file.

## Solution
Add an output cap constant and enforce it in all 4 command substitution paths.

## Steps
1. Add a constant at the top of `src/executor/expansion.rs`:
   ```rust
   /// Maximum bytes captured from a command substitution before truncation.
   /// Configurable via RUSH_MAX_SUBST_OUTPUT env var. Default: 50MB.
   const DEFAULT_MAX_SUBSTITUTION_OUTPUT: usize = 50 * 1024 * 1024;

   fn max_substitution_output() -> usize {
       std::env::var("RUSH_MAX_SUBST_OUTPUT")
           .ok()
           .and_then(|s| crate::run_api::parse_max_output(&s))
           .unwrap_or(DEFAULT_MAX_SUBSTITUTION_OUTPUT)
   }
   ```

2. In `execute_command_substitution()` (around line 738), after `let result = sub_executor.execute(statements)?;`, add:
   ```rust
   let mut output = result.stdout();
   let max = max_substitution_output();
   if output.len() > max {
       output.truncate(max);
       // Truncate at UTF-8 boundary
       while !output.is_char_boundary(output.len()) {
           output.pop();
       }
       eprintln!("rush: warning: command substitution output truncated at {} bytes", max);
   }
   Ok(output.trim_end().to_string())
   ```

3. Apply the same pattern in `execute_command_substitution_str()` (around line 680).

4. Apply the same pattern in the two `_static` variants: `resolve_argument_static()` (around line 930) and `expand_command_substitutions_in_string_static()` (around line 1020 and 1060 — there are two places inside this function that call `sub_executor.execute()`).

5. Add a test in `src/executor/expansion.rs` or a test file:
   ```rust
   #[cfg(test)]
   mod substitution_cap_tests {
       use super::*;
       
       #[test]
       fn test_command_substitution_output_cap() {
           // Verify the constant exists and has a reasonable default
           assert!(DEFAULT_MAX_SUBSTITUTION_OUTPUT >= 1024 * 1024); // at least 1MB
           assert!(DEFAULT_MAX_SUBSTITUTION_OUTPUT <= 100 * 1024 * 1024); // at most 100MB
           
           // Verify the function respects env var
           std::env::set_var("RUSH_MAX_SUBST_OUTPUT", "1KB");
           assert_eq!(max_substitution_output(), 1024);
           std::env::remove_var("RUSH_MAX_SUBST_OUTPUT");
       }
   }
   ```
