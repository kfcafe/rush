## Task

Add support for file descriptor duplication redirects. Currently only `2>&1` works 
(hardcoded as `StderrToStdout` token). Need general `N>&M` and `>&N` support.

**Missing patterns:**
- `>&2` — redirect stdout to stderr (shorthand for `1>&2`)
- `1>&2` — explicit stdout to stderr  
- `N>&M` — duplicate fd N to fd M (general case)
- `>&-` — close stdout
- `N>&-` — close fd N

## Step 1: Lexer (src/lexer/mod.rs)
Add new token variants or generalize the redirect tokens:
- `>&2` is NOT `BothRedirect` (`&>`) — it is stdout-to-fd duplication
- Need to handle `N>&M` where N and M are digits

Options:
a) Add specific tokens: `StdoutToStderr` for `>&2` / `1>&2`
b) Add generic `FdDup(u32, u32)` token for `N>&M`
c) Keep `>&2` as a token pattern like `StderrToStdout`

Recommend option (c) — add `#[token(">&2")] StdoutToStderr` and `#[token("1>&2")] StdoutToStderr`

## Step 2: AST (src/parser/ast.rs)
Add to RedirectKind:
```rust
StdoutToStderr,  // >&2 or 1>&2
```

## Step 3: Parser (src/parser/mod.rs)
Handle the new token in `parse_command` and `parse_single_redirect`.

## Step 4: Executor (src/executor/mod.rs)
Wire up the fd duplication using `dup2()`. When executing a command with 
`StdoutToStderr` redirect, duplicate fd 2 onto fd 1 before exec.
