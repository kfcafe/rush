## Task

The `echo` builtin ignores flags `-e` and `-n`, printing them literally:
- `echo -e "a\nb"` outputs `-e a\nb` instead of `a` + newline + `b`
- `echo -n "test"` outputs `-n test\n` instead of `test` (no trailing newline)

## Implementation

### echo -n
Suppress the trailing newline.

### echo -e  
Process escape sequences: `\n` → newline, `\t` → tab, `\\` → backslash, 
`\a` → bell, `\b` → backspace, `\r` → carriage return, `\0NNN` → octal, `\xHH` → hex.

### Combined flags
`echo -en "test\n"` — both flags together.

## Files
- src/builtins/ (find and modify echo implementation)  
- src/executor/mod.rs (if echo is handled inline, modify there)

## Acceptance
- `rush -c 'echo -e "a\nb"'` outputs two lines: "a" and "b"
- `rush -c 'echo -n test'` outputs "test" with no trailing newline
- `cargo test` passes
