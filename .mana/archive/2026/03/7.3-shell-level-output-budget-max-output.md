Add shell-level output size limiting so callers don't need to truncate after the fact.

## Why

Tower's imp agent engine currently does manual output truncation in its BashTool (MAX_OUTPUT_LINES=2000, MAX_OUTPUT_BYTES=50KB). This is wasteful — the command runs to completion producing potentially megabytes of output, then imp throws most of it away. Rush should be able to cap output at the source.

## What to build

### CLI flag
```bash
rush -c "find / -name '*.rs'" --max-output 50KB
rush -c "cat huge_file.log" --max-output 2000lines
```

### Env var
```bash
RUSH_MAX_OUTPUT=50KB rush -c "find / -name '*.rs'"
```

### Library API (already in RunOptions from 7.1)
```rust
rush::run("find / -name '*.rs'", &RunOptions {
    max_output_bytes: Some(50 * 1024),
    ..Default::default()
})
```

### Behavior:
- Applies to stdout only (stderr is typically small)
- When the limit is reached, stop accumulating output and append a truncation notice: `\n[Output truncated: {bytes_written} bytes, limit {max_bytes}]`
- For `max_output_bytes`: count bytes, stop when exceeded
- The `RunResult::truncated` field (from 7.1) should be set to true
- For built-in commands: check the limit during output generation, stop early if possible (e.g., ls can stop listing files, grep can stop searching)
- For external commands: monitor the pipe, stop reading when limit is reached (but let the process finish to get its exit code)

### Parsing the --max-output value:
- `50KB`, `50kb`, `50k` → 50 * 1024 bytes
- `1MB`, `1mb`, `1m` → 1 * 1024 * 1024 bytes
- `2000lines`, `2000l` → count newlines (convert to approximate byte budget or count lines directly)
- Plain number → bytes

## Files to modify
- `src/main.rs` — add `--max-output` CLI flag, parse it
- `src/run_api.rs` — apply max_output_bytes from RunOptions in the run() function
- `src/executor/mod.rs` — add max_output tracking to ExecutionResult accumulation (optional optimization for built-ins)
- `src/runtime/mod.rs` — store max_output setting if you want built-ins to check it

## Tests to write (all prefixed test_output_budget_)
- `test_output_budget_basic` — run a command producing 10KB with max_output_bytes=1024, verify output.len() <= 1024 + truncation notice length, verify truncated=true
- `test_output_budget_no_truncation` — run a command producing 100 bytes with max_output_bytes=1024, verify truncated=false
- `test_output_budget_via_env` — set RUSH_MAX_OUTPUT=1KB env var, run command, verify truncation
- `test_output_budget_truncation_notice` — verify the truncation notice text is present when truncated
- `test_output_budget_external_cmd` — external command (like /bin/echo or seq) with output budget, verify it still works and truncates
