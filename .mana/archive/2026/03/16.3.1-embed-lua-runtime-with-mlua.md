## Task

Embed the Lua runtime using `mlua` crate. Set up the core bridge between Rust and Lua: expose rush's Value types, allow Lua scripts to register builtins, and load user scripts from `~/.rush/lua/`.

## Design

### Rust → Lua bridge (src/lua/mod.rs)
```rust
pub struct LuaRuntime {
    lua: mlua::Lua,
}

impl LuaRuntime {
    pub fn new() -> Result<Self>;
    pub fn load_user_scripts(&self) -> Result<()>;  // ~/.rush/lua/*.lua
    pub fn call_hook(&self, name: &str, args: &[Value]) -> Result<()>;
    pub fn get_registered_builtins(&self) -> Vec<LuaBuiltin>;
}
```

### Lua API exposed to scripts
```lua
-- Register a custom builtin
rush.register_builtin("weather", {
    description = "Show weather for a city",
    run = function(args)
        local city = args[1] or "NYC"
        local result = rush.exec("curl -s wttr.in/" .. city .. "?format=j1")
        return rush.json_parse(result)  -- returns structured data
    end
})

-- Register a prompt segment
rush.register_prompt("git_status", function()
    local branch = rush.exec("git branch --show-current 2>/dev/null")
    if branch ~= "" then
        return { text = "(" .. branch .. ")", color = "yellow" }
    end
end)

-- Hook into precmd/preexec
rush.on("precmd", function(exit_code, elapsed_ms)
    -- custom logic
end)
```

### Key APIs to expose
- `rush.exec(cmd)` — run a shell command, return stdout as string
- `rush.exec_structured(cmd)` — run a rush builtin, return structured data as Lua table
- `rush.json_parse(str)` — parse JSON string to Lua table
- `rush.json_encode(table)` — encode Lua table to JSON string
- `rush.env.get(name)` / `rush.env.set(name, value)`
- `rush.cwd()` — current working directory
- `rush.register_builtin(name, spec)`
- `rush.register_prompt(name, fn)`
- `rush.register_completion(name, fn)`
- `rush.on(event, fn)` — hook into shell events

## Cargo.toml
```toml
mlua = { version = "0.10", features = ["lua54", "vendored"] }
```

The `vendored` feature statically links Lua — no system dependency.
