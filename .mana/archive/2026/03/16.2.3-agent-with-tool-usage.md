## Task

Rewrite the `?` feature from a one-shot command generator into a real agent with tool usage. The agent can run commands, read files, write files, and make surgical edits — with confirmation before destructive operations.

## Tools

Four tools, same primitives as effective coding agents:

### `shell` — run a command
```json
{
  "name": "shell",
  "description": "Execute a shell command. Returns stdout, stderr, and exit code. User confirms before execution.",
  "parameters": {
    "command": "string (required) — the shell command to run"
  }
}
```

### `read` — read a file
```json
{
  "name": "read",
  "description": "Read the contents of a file. No confirmation needed (read-only).",
  "parameters": {
    "path": "string (required) — file path to read"
  }
}
```

### `write` — write/create a file
```json
{
  "name": "write",
  "description": "Write content to a file. Creates the file if it does not exist, overwrites if it does. User confirms before write. Automatically tracked by rush undo system.",
  "parameters": {
    "path": "string (required) — file path",
    "content": "string (required) — file content"
  }
}
```

### `edit` — surgical find-and-replace
```json
{
  "name": "edit",
  "description": "Edit a file by replacing exact text. oldText must match exactly. User confirms before edit. Automatically tracked by rush undo system.",
  "parameters": {
    "path": "string (required) — file path",
    "old_text": "string (required) — exact text to find",
    "new_text": "string (required) — replacement text"
  }
}
```

## Confirmation UX

```
? add error handling to src/main.rs

  📖 read src/main.rs
  
  ✏️  edit src/main.rs
  - fn main() {
  + fn main() -> Result<()> {
  Apply? [Y/n] y
  ✓ Applied (undo available)

  ⚡ shell: cargo check
  Run? [Y/n] y
  ✓ No errors

  Done. Added Result return type to main().
  Undo with: undo
```

Rules:
- `read` — auto-executes, no confirmation (read-only)
- `shell` — shows command, asks confirmation
- `write` — shows path + content preview, asks confirmation
- `edit` — shows diff-style preview, asks confirmation
- All file mutations go through rush undo system automatically
- User can Ctrl-C at any point to abort

## Agent loop

```rust
pub struct Agent {
    client: LlmClient,
    tools: Vec<Tool>,
}

impl Agent {
    pub fn run(&mut self, intent: &str, runtime: &mut Runtime) -> Result<()> {
        let cwd = runtime.get_cwd().display().to_string();
        let mut messages = vec![
            Message::system(format!(
                "You are a shell assistant in the Rush shell.\n\
                 Current directory: {cwd}\n\
                 You have tools: shell, read, write, edit.\n\
                 Be concise — this is a terminal.\n\
                 Inspect before acting. Explain before modifying.\n\
                 For file changes, use edit for surgical changes, write for new files."
            )),
            Message::user(intent.to_string()),
        ];

        loop {
            let response = self.client.chat(&messages, Some(&self.tools))?;
            match response {
                Response::Text(text) => {
                    println!("{}", text);
                    break;
                }
                Response::ToolCall { name, arguments } => {
                    let result = self.execute_tool(&name, &arguments, runtime)?;
                    messages.push(Message::assistant_tool_call(&name, &arguments));
                    messages.push(Message::tool_result(&name, &result));
                }
            }
        }
        Ok(())
    }

    fn execute_tool(&self, name: &str, args: &serde_json::Value, runtime: &mut Runtime) -> Result<String> {
        match name {
            "read" => {
                let path = args["path"].as_str().unwrap();
                println!("  📖 read {}", path);
                std::fs::read_to_string(path).map_err(Into::into)
            }
            "shell" => {
                let cmd = args["command"].as_str().unwrap();
                println!("  ⚡ shell: {}", cmd);
                // Ask confirmation
                if !confirm("  Run?")? { return Ok("User declined".into()); }
                // Execute via rush executor for structured output from builtins
                execute_command(cmd, runtime)
            }
            "write" => {
                let path = args["path"].as_str().unwrap();
                let content = args["content"].as_str().unwrap();
                println!("  📝 write {} ({} bytes)", path, content.len());
                if !confirm("  Write?")? { return Ok("User declined".into()); }
                // Track in undo system
                runtime.undo_manager_mut().track_write(path)?;
                std::fs::write(path, content)?;
                Ok("Written".into())
            }
            "edit" => {
                let path = args["path"].as_str().unwrap();
                let old = args["old_text"].as_str().unwrap();
                let new = args["new_text"].as_str().unwrap();
                // Show diff-style preview
                println!("  ✏️  edit {}", path);
                show_edit_preview(old, new);
                if !confirm("  Apply?")? { return Ok("User declined".into()); }
                // Track in undo system
                runtime.undo_manager_mut().track_modify(path)?;
                let content = std::fs::read_to_string(path)?;
                let updated = content.replace(old, new);
                std::fs::write(path, updated)?;
                Ok("Applied".into())
            }
            _ => Ok(format!("Unknown tool: {}", name)),
        }
    }
}
```

## System prompt

Keep it minimal. The agent knows the cwd, has 4 tools, and should:
- Read before writing
- Show what it is doing to do before doing it
- Prefer edit over write for existing files
- Use shell to verify changes (cargo check, tests, etc.)
