## Task
Fix \$ and \` inside double-quoted strings so they produce literal characters
that are NOT expanded as variables or command substitutions.

## The bug
Currently: `"\$VAR"` → parser does process_double_quote_escapes → produces "$VAR" as
Argument::Literal → executor calls expand_variables_in_literal → expands $VAR.
The escape was processed but the expansion undoes it.

The fundamental issue: after escape processing, there is no way to distinguish
"this $ came from \$ (literal)" vs "this $ is a variable reference (expand)".

## Solution
Use Argument::DoubleQuoted(Vec<ArgumentPart>) from the AST changes (bean 6.1.1).
Parse the double-quoted content into parts where escaped characters become
ArgumentPart::Literal and variable references become ArgumentPart::Variable.

## Steps
1. Write `fn parse_double_quoted_content(content: &str) -> Vec<ArgumentPart>` in parser.
   Scan character by character:
   - `\$` → ArgumentPart::Literal("$")
   - `\`` → ArgumentPart::Literal("`")
   - `\\` → ArgumentPart::Literal("\\")
   - `\"` → ArgumentPart::Literal("\"")
   - `$VARNAME` → ArgumentPart::Variable("$VARNAME")
   - `${...}` → ArgumentPart::BracedVariable("${...}")
   - `$(...)` → ArgumentPart::CommandSubstitution("$(...)")
   - `` `...` `` → ArgumentPart::CommandSubstitution("`...`")
   - plain text → ArgumentPart::Literal("text")
   Merge adjacent Literal parts for cleanliness.

2. Change parse_argument Token::String handling:
   FROM: strip quotes → process_double_quote_escapes → Argument::Literal
   TO:   strip quotes → parse_double_quoted_content → Argument::DoubleQuoted(parts)

3. In executor resolve_argument functions, add:
   ```rust
   Argument::DoubleQuoted(parts) => {
       let mut result = String::new();
       for part in parts {
           match part {
               ArgumentPart::Literal(s) => result.push_str(s),
               ArgumentPart::Variable(v) => {
                   // reuse existing variable expansion logic
                   let expanded = self.resolve_argument(&Argument::Variable(v.clone()))?;
                   result.push_str(&expanded);
               }
               ArgumentPart::BracedVariable(v) => {
                   let expanded = self.resolve_argument(&Argument::BracedVariable(v.clone()))?;
                   result.push_str(&expanded);
               }
               ArgumentPart::CommandSubstitution(cmd) => {
                   let expanded = self.resolve_argument(&Argument::CommandSubstitution(cmd.clone()))?;
                   result.push_str(&expanded);
               }
           }
       }
       Ok(result)
   }
   ```
   Add this to ALL resolve_argument functions (expansion.rs, mod.rs, pipeline.rs).

4. Update ALL other Token::String handlers in the parser (assignments, expressions,
   redirects, case patterns) to either use DoubleQuoted or at minimum parse parts
   and stringify.

## POSIX 2.2.3 escape rule
In double quotes, backslash retains special meaning ONLY before: $ ` " \ newline.
Before any other char, the backslash is preserved literally.

## Fixes these 3 tests
- test_double_quotes_escape_dollar ("\$VAR" → literal $VAR)
- test_double_quotes_escape_backtick ("\`cmd\`" → literal backticks)
- test_quote_escaping_with_variable ("\\$VAR" → backslash + expanded value)
