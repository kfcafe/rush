Add a global agent mode that makes rush's output structured/JSON by default for all built-in commands that support it. This eliminates the need to pass `--json` to every individual command when rush is being used programmatically.

## What to build

When `RUSH_AGENT_MODE=1` is set (or `RunOptions::json_output` is true in the library API), built-in commands that have JSON output capability use it automatically.

### Behavior when agent mode is active:
1. Built-in commands (ls, grep, find, git status, etc.) emit JSON instead of formatted text
2. ANSI color codes are stripped from ALL output (both built-in and external commands)
3. External commands run normally — their output can't be made structured, but ANSI stripping still applies

### Detection:
- Check `std::env::var("RUSH_AGENT_MODE")` at Runtime initialization
- Store as `agent_mode: bool` on `Runtime`
- In `run()` API: set `agent_mode = true` when `RunOptions::json_output == true`
- Built-in implementations check `runtime.agent_mode` and act as if `--json` was passed

### Which built-ins to update:
Look at which built-ins already have `--json` handling. For each, add a check at the top:
```rust
let json_mode = args.contains("--json") || runtime.agent_mode;
```

Likely candidates (check each file for existing --json support):
- `src/builtins/ls.rs`
- `src/builtins/grep.rs`
- `src/builtins/find.rs`
- `src/builtins/git_status.rs`
- `src/builtins/git_log.rs`

Commands like `echo`, `cd`, `export` don't need JSON — their output is already simple text.

### ANSI stripping:
Add a helper in `src/output/mod.rs` (or similar) that strips ANSI escape sequences from a string. Apply it to all output when agent_mode is true.

## Files to modify
- `src/runtime/mod.rs` — add `agent_mode: bool` field, check env var on init, add `pub fn agent_mode(&self) -> bool`
- `src/builtins/ls.rs` — check runtime.agent_mode
- `src/builtins/grep.rs` — check runtime.agent_mode
- `src/builtins/find.rs` — check runtime.agent_mode
- Other built-ins with --json support — check runtime.agent_mode
- `src/run_api.rs` — set runtime.agent_mode when RunOptions::json_output is true
- `src/output/mod.rs` — add ANSI stripping helper

## Tests to write (all prefixed test_agent_mode_)
- `test_agent_mode_ls_json` — with RUSH_AGENT_MODE=1, `ls /tmp` output parses as valid JSON array
- `test_agent_mode_via_run_api` — `rush::run("ls /tmp", &RunOptions { json_output: true, .. })` returns valid JSON
- `test_agent_mode_no_ansi` — with agent mode, output from commands contains zero ANSI escape sequences (regex check for \x1b\[)
- `test_agent_mode_external_passthrough` — external commands still work, their text output is returned (just ANSI-stripped)
- `test_agent_mode_echo_unchanged` — `echo hello` still returns plain "hello\n" (not JSON-wrapped)
