## Task

Convert the key builtins from emitting text (or JSON-as-text behind `--json`) to emitting `Output::Structured` by default. The `--json` flag becomes redundant — structured output is the native format.

## Builtins to convert (priority order)
1. `ls` — already has JSON serialization, convert to Output::Structured(serde_json::Value)
2. `find` — same pattern
3. `grep` — has GrepMatch struct, emit as structured
4. `git status` — has structured data internally
5. `git log` — has structured data internally
6. `git branch` — has structured data internally
7. `ps`/`jobs` — process/job listings

## Approach

Each builtin currently does:
```rust
let json_output = serde_json::to_string_pretty(&entries)?;
Ok(ExecutionResult { output: Output::Text(json_output), ... })
```

Change to:
```rust
let json_value = serde_json::to_value(&entries)?;
Ok(ExecutionResult { output: Output::Structured(json_value), ... })
```

The `Output::Structured` variant already exists. The `as_text()` method already pretty-prints it. This is mostly mechanical.

## Key constraint
- Do NOT remove the `--json` flag yet — keep it for backward compat, just make it a no-op when output is already structured
- Text rendering (the pretty table/column output) moves to the auto-render layer (separate unit)
- External commands piped after a structured builtin should receive text (handled by pipeline unit)
