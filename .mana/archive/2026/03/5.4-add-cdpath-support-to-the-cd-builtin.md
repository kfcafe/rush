## Task
When `cd dirname` fails because the path doesn't exist relative to cwd, search directories listed in the `CDPATH` variable (colon-separated) before giving up.

POSIX behavior: if the argument doesn't start with `/`, `./`, or `../`, and CDPATH is set, try each directory in CDPATH as a prefix. If found, cd there and print the resolved path to stdout (POSIX requirement — printing lets the user know which CDPATH entry was used).

## Steps
1. In `src/builtins/mod.rs`, in `builtin_cd`:
   - After computing `absolute` and before the `if !absolute.exists()` check, add CDPATH logic.
   - Only trigger CDPATH if: the original path argument doesn't start with `/`, `./`, or `../`, AND the resolved path doesn't exist.
   - Read `CDPATH` from `runtime.get_variable("CDPATH")`.
   - Split by `:` and for each directory, check if `cdpath_dir/arg` exists and is a directory.
   - If found, use that path instead. Print the resolved path to stdout (like `cd -` does).
   - If no CDPATH match either, fall through to the existing error handling.
2. Add tests in a new test file or inline tests that verify:
   - CDPATH with single directory works
   - CDPATH with multiple directories picks the first match
   - CDPATH is not used when path starts with `/`, `./`, or `../`
   - CDPATH prints the resolved path to stdout
