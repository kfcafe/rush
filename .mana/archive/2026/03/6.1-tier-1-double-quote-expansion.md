## Problem
Double-quoted strings are treated as literals. Variables, command substitutions, and
escape sequences inside "..." are not expanded. This breaks virtually all real-world
shell usage: "$HOME", "$(cmd)", "value: $VAR", etc.

## Scope
19 of 38 quoting tests currently fail. All 38 must pass when this tier is done.

## Current behavior
- `echo "$HOME"` prints literal "$HOME" instead of /Users/asher
- `echo "$(pwd)"` prints literal "$(pwd)"
- `echo "quote: \"hello\""` prints literal backslash sequences

## Root cause
Parser treats Token::String identically to Token::SingleQuotedString —
both produce Argument::Literal with no expansion.

## Files involved
- src/parser/ast.rs — Argument enum needs DoubleQuoted variant
- src/parser/mod.rs — parse_argument and 6+ other Token::String handlers
- src/executor/expansion.rs — resolve_argument needs DoubleQuoted handling
- src/executor/mod.rs — resolve_argument (duplicate) needs same
- src/executor/pipeline.rs — resolve_argument (duplicate) needs same
- tests/quoting_tests.rs — 38 tests, 19 currently failing
