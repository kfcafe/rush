## Task
Implement `head` and `tail` as builtins — pipeline staples, low complexity.

## Behavior
- `head [-n N] [FILE...]` — print first N lines (default 10)
- `head -c N [FILE...]` — print first N bytes
- `tail [-n N] [FILE...]` — print last N lines (default 10)
- `tail -c N [FILE...]` — print last N bytes
- `tail -n +N` — print from line N onward
- Both read from stdin if no file given (pipeline support)
- `tail -f` (follow) is optional / stretch goal — can defer to external for -f

## Implementation
1. Create `src/builtins/head.rs` and `src/builtins/tail.rs`
2. head: simple BufReader line iteration with early exit
3. tail: read all lines into VecDeque, output last N (or seek from end for -c)
4. Both must support stdin via `execute_with_stdin` pattern (see cat/grep)
5. Register in mod.rs
