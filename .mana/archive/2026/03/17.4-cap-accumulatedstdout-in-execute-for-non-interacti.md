## Problem
`Executor::execute()` in `src/executor/mod.rs` accumulates all stdout from all statements into `accumulated_stdout: String` with no size limit:
```rust
accumulated_stdout.push_str(&result.stdout());
```
When called from a command substitution context, a long script producing lots of output will buffer everything into one giant String.

## Solution
Add a check after each push_str that caps `accumulated_stdout` at a configurable limit. Only warn, don't error — truncate silently after printing a warning.

## Steps
1. In `Executor::execute()` (around line 120 of `src/executor/mod.rs`), after the line:
   ```rust
   accumulated_stdout.push_str(&result.stdout());
   ```
   Add:
   ```rust
   // Cap accumulated output to prevent unbounded memory growth.
   // This matters most in command substitution contexts where all
   // output is captured into a String.
   const MAX_ACCUMULATED: usize = 50 * 1024 * 1024; // 50MB
   if accumulated_stdout.len() > MAX_ACCUMULATED {
       accumulated_stdout.truncate(MAX_ACCUMULATED);
       // Ensure we're at a UTF-8 boundary
       while !accumulated_stdout.is_char_boundary(accumulated_stdout.len()) {
           accumulated_stdout.pop();
       }
       // Only warn once
       if accumulated_stderr.is_empty() || !accumulated_stderr.contains("output truncated") {
           accumulated_stderr.push_str("rush: warning: accumulated output truncated at 50MB\n");
       }
   }
   ```

2. Verify it compiles: `cargo check`

## Files
- `src/executor/mod.rs` (modify — add cap after push_str in execute())

## Don't
- Don't change `execute_statement()` or any other methods
- Don't change how interactive output works (the REPL prints each command's output immediately, this only affects buffered execution like scripts and substitutions)
- Don't add env var configurability here — the substitution cap from unit 17.1 covers the env var case
