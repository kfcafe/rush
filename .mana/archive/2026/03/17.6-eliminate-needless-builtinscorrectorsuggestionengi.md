## Problem
The static command substitution functions in `src/executor/expansion.rs` create brand new `Builtins::new()`, `Corrector::new()`, `SuggestionEngine::new()`, and `TerminalControl::new()` for every substitution. There are 3 places that do this inside:
- `resolve_argument_static()` (~line 925)
- `expand_command_substitutions_in_string_static()` (~lines 1020 and 1065 — two sub-executor creations)

These objects are constructed and thrown away after each substitution.

## Solution
Change the static functions to accept the heavy objects as parameters instead of constructing new ones. The caller (`execute_parallel` in `src/executor/mod.rs`) already has `Arc<Builtins>` and `Arc<Corrector>` — thread them through.

## Steps

### 1. Change function signatures in `src/executor/expansion.rs`

```rust
// Before:
pub(crate) fn resolve_argument_static(arg: &Argument, runtime: &Runtime) -> String

// After:
pub(crate) fn resolve_argument_static(
    arg: &Argument,
    runtime: &Runtime,
    builtins: &Builtins,
    corrector: &Corrector,
) -> String
```

```rust
// Before:
pub(crate) fn expand_command_substitutions_in_string_static(input: &str, runtime: &Runtime) -> String

// After:
pub(crate) fn expand_command_substitutions_in_string_static(
    input: &str,
    runtime: &Runtime,
    builtins: &Builtins,
    corrector: &Corrector,
) -> String
```

### 2. Update the sub-executor creation inside these functions

Replace all 3 occurrences of:
```rust
builtins: Builtins::new(),
corrector: Corrector::new(),
```
With:
```rust
builtins: builtins.clone(),
corrector: corrector.clone(),
```

### 3. Update `expand_and_resolve_arguments_static()` signature too

It calls `resolve_argument_static`, so it needs the same extra params:
```rust
pub(crate) fn expand_and_resolve_arguments_static(
    args: &[Argument],
    runtime: &Runtime,
    builtins: &Builtins,
    corrector: &Corrector,
) -> Result<Vec<String>>
```

And pass them through to `resolve_argument_static()`.

### 4. Update all callers

Search for `expand_and_resolve_arguments_static` and `resolve_argument_static` across the entire `src/` and update all call sites to pass builtins and corrector refs.

In `execute_parallel()` (src/executor/mod.rs ~line 1525), the caller already has `Arc<Builtins>` and `Arc<Corrector>`, so just pass `&builtins` and `&corrector`.

Check `src/executor/pipeline.rs` too.

### 5. Verify: `cargo check`
