## Task

Interactive setup wizard that runs on first `?` usage (or via `rush --setup-ai`). Auto-detects available backends and guides the user through configuration.

## Flow

```
rush: No AI backend configured for ? queries.

  1) Ollama (local, private, recommended)
  2) OpenAI API (requires key)
  3) Anthropic API (requires key)
  4) Skip for now

> 1
Checking Ollama... found at localhost:11434
Available models: llama3.2 (3B), qwen2.5-coder (7B), codellama (13B)
Recommended for shell commands: qwen2.5-coder (7B)

Install qwen2.5-coder? [Y/n] y
Pulling qwen2.5-coder:7b... done (4.1 GB)

✓ AI configured. Try: ? find all rust files modified today
Config saved to ~/.rush/ai.toml
```

## Auto-detection
1. Check if Ollama is running: GET `http://localhost:11434/api/tags`
2. If running, list installed models
3. If a suitable model is installed, offer to use it
4. If no suitable model, offer to pull one (via `ollama pull`)
5. If Ollama not running, check for API keys in env (OPENAI_API_KEY, ANTHROPIC_API_KEY)

## Implementation
- `src/ai/wizard.rs` — the interactive wizard
- Triggered from `src/intent/mod.rs` when no config exists
- Also callable via `rush --setup-ai` flag in main.rs
- Writes config to `~/.rush/ai.toml`

- src/ai/wizard.rs (create)
- src/ai/config.rs (modify — add load/save to ~/.rush/ai.toml)
- src/intent/mod.rs (modify — trigger wizard when no config)
- src/main.rs (modify — add --setup-ai flag)
