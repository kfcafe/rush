## Task
Enhance the `RushPrompt` to show git branch info and last exit code, making it useful without customization.

Current prompt: `~/rush> `
New prompt: `~/rush (main)> ` (in a git repo with clean status)
After failed command: `~/rush (main) [1]> ` (shows non-zero exit code)

## Steps
1. Make `RushPrompt` hold state: `last_exit_code: i32` (default 0).
2. In `get_prompt_indicator`, after building the cwd string:
   - Detect git branch by reading `.git/HEAD` (walk up from cwd until found). Parse `ref: refs/heads/{branch}` to get the branch name. If detached, show first 7 chars of the hash.
   - If in a git repo, append ` ({branch})` in a muted color (e.g. light purple or yellow).
   - If `last_exit_code != 0`, append ` [{code}]` in red.
3. Update the REPL loop to set `prompt.last_exit_code` after each command execution, before the next iteration.
4. The `RushPrompt` will need to become a mutable struct (use `Cell<i32>` or `AtomicI32` for the exit code since reedline borrows it immutably via the `Prompt` trait).

## Context
```rust
// Current prompt struct (src/main.rs):

    fn new() -> Self { Self }
    fn get_prompt_indicator(&self) -> String {
        let cwd = /* ... shortened path ... */;

```

The git module exists at `src/git/mod.rs` with a `GitContext` struct, but it does a full `git status` which is too slow for every prompt. Instead, just read `.git/HEAD` directly — it's a single file read, effectively free.

## Don't
- Don't use `git` subprocess or `GitContext` — too slow for prompts. Just read `.git/HEAD`.
- Don't add external dependencies for this
- Don't break the existing prompt format for non-git directories (should still show `~/path> `)
- Keep it lightweight — this runs before every single prompt
