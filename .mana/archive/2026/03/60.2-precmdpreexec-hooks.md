## Task
Implement `precmd` and `preexec` hooks — functions that run before displaying the prompt and before executing each command. Needed for prompt themes, timing display, directory tracking, etc.

## Behavior
- `precmd` functions run after each command completes, before showing the next prompt
- `preexec` functions run just before a command is executed (after parsing, before exec)
- Users register hooks via: defining functions named `rush_precmd` / `rush_preexec`, or an API like `trap ... PROMPT`
- Hooks receive context: last exit code, command string, execution time

## Files
- `src/executor/mod.rs` or `src/runtime/mod.rs` (modify — add hook invocation points)
- May need `src/hooks.rs` for hook management
