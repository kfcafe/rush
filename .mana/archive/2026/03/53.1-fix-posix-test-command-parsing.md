## Task

The `[` command (POSIX test) fails to parse. `test -f /etc/hosts` works but `[ -f /etc/hosts ]` does not.

**Error:** `rush: Invalid token at position 0: "[ -f /etc/hosts "`

The issue is in the **lexer**. The `[` token is `LeftBracket` but something is consuming
`[ -f /etc/hosts ]` as a single invalid token instead of tokenizing it as
`LeftBracket` + `ShortFlag(-f)` + `Path(/etc/hosts)` + `RightBracket`.

The GlobPattern regex may be interfering:
```rust
#[regex(r"[a-zA-Z0-9_.\-/]*\[[^\]]+\][a-zA-Z0-9_.*?\-/]+", |lex| lex.slice().to_string())]
GlobPattern(String),
```

This regex matches `prefix[content]suffix` — but the `[` token should take priority 
when `[` appears as a standalone command (with whitespace after it).
