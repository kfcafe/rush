## Task
Emit OSC 133 escape sequences to mark prompt boundaries and command output regions. This enables modern terminals (iTerm2, Terminal.app Sonoma+, WezTerm) to:
- Jump between prompts with Cmd+Shift+Up/Down
- Click to select entire command output
- Visually separate commands

## Protocol
OSC 133 uses four markers:
- `\x1b]133;A\x07` — Prompt start (before the prompt is drawn)
- `\x1b]133;B\x07` — Prompt end / command start (after user presses Enter)
- `\x1b]133;C\x07` — Command output start (before output begins)
- `\x1b]133;D;{exit_code}\x07` — Command finished (with exit code)

## Steps
1. In `src/terminal.rs`, add four public functions:
   - `pub fn mark_prompt_start()` — writes `\x1b]133;A\x07` to stderr
   - `pub fn mark_command_start()` — writes `\x1b]133;B\x07` to stderr
   - `pub fn mark_output_start()` — writes `\x1b]133;C\x07` to stderr
   - `pub fn mark_command_finished(exit_code: i32)` — writes `\x1b]133;D;{exit_code}\x07` to stderr

2. In `src/main.rs`, in `run_interactive_with_reedline`, add calls at the right points in the REPL loop:
   - `mark_prompt_start()` — right before `line_editor.read_line(&prompt)` (before the existing `emit_osc7_cwd()` call)
   - `mark_command_start()` and `mark_output_start()` — after a line is read successfully, before execution begins
   - `mark_command_finished(exit_code)` — after command execution completes (both success and error paths), using the actual exit code

3. Add tests that verify the escape sequences are correctly formatted.

## Context
The REPL loop structure in main.rs:

- Don't wrap these in any feature flag — all modern terminals either use them or ignore them harmlessly
