## Task

Nested `$()` inside double-quoted strings is broken:
- `echo "$(echo "$(echo deep)")"` outputs `$(echo  deep )` instead of `deep`
- `echo $(echo $(echo deep))` works fine (unquoted)

The issue is in the **lexer**. When parsing a double-quoted string that contains
`$(...)`, the nested quotes confuse the tokenizer. It stops at the inner `"` 
instead of tracking the `$()` nesting depth.

## Root Cause

The String regex in the lexer:
```rust
#[regex(r#""([^"\\]|\\.)*""#, |lex| lex.slice().to_string())]
String(String),
```

This regex matches `"..."` greedily but cannot handle `"$(echo "nested")"` because
the inner `"` terminates the outer string match.

## Fix Options

1. **Custom string parser**: Replace the regex with a callback that tracks `$()` depth
   inside double-quoted strings, similar to `parse_command_substitution`
2. **Two-pass approach**: Tokenize the outer string, then re-tokenize `$(...)` contents

Option 1 is more robust.
