## Task
Set the terminal tab title to the current directory (e.g. `~/rush`) before each prompt, and to the running command (e.g. `vim main.rs`) when a command starts executing. This uses OSC 0 and/or OSC 2 escape sequences.

## How it works
- **Before prompt**: emit `\x1b]2;rush: ~/path\x07` to stderr — shows the directory in the tab title.
- **Before command execution**: emit `\x1b]2;command-text\x07` to stderr — shows the running command in the tab title.

OSC 2 sets the window title. OSC 0 sets both window and icon title. Either works; OSC 2 is more standard.

## Steps
1. In `src/terminal.rs`, add a `pub fn set_terminal_title(title: &str)` function that writes `\x1b]2;{title}\x07` to stderr.
2. In `src/terminal.rs`, add a `pub fn set_terminal_title_to_cwd()` function that formats the cwd with `~` shortening (like `RushPrompt::get_prompt_indicator` does) and calls `set_terminal_title` with `"rush: {path}"`.
3. In `src/main.rs`, in the REPL loop:
   - Call `terminal::set_terminal_title_to_cwd()` right next to the existing `terminal::emit_osc7_cwd()` call (before the prompt).
   - After `line_editor.read_line(&prompt)` succeeds with a non-empty line, call `terminal::set_terminal_title(line)` before executing the command — so the tab shows the running command.
4. Add unit tests for `set_terminal_title` (verify it produces the right escape sequence — can capture stderr or just test the formatting helper).
