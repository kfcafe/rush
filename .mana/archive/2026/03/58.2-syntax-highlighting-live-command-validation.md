## Task
Add live syntax highlighting: valid commands show green, invalid show red, strings show yellow, etc. Fish and nushell both have this — it's table stakes for modern shells.

## Implementation
Reedline supports custom highlighters via the `Highlighter` trait:
1. Implement `reedline::Highlighter` for Rush
2. In `highlight(&self, line: &str)` method:
   - Tokenize the line using Rush's lexer
   - Color command names: green if builtin/exists in PATH, red if not found
   - Color strings/quotes: yellow
   - Color pipes/operators: cyan
   - Color comments: grey
   - Color flags: default
3. Wire into reedline: `.with_highlighter(Box::new(RushHighlighter::new(...)))`

## Context
```rust
impl Highlighter for RushHighlighter {
    fn highlight(&self, line: &str, _cursor: usize) -> StyledText { ... }
}
```

## Files
- `src/highlight.rs` (create — RushHighlighter implementing reedline::Highlighter)

## Don't
- Don't implement a full parser for highlighting — tokenization is enough
- Don't slow down input — highlighting must be fast (< 1ms)
