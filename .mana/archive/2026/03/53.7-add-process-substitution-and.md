## Task

Add process substitution support: `<(command)` and `>(command)`.

- `<(cmd)` runs cmd and provides its stdout as a file path (named pipe/fd)
- `>(cmd)` runs cmd and provides its stdin as a file path

**Current error:** `rush: Expected redirect target`

Common uses:
- `diff <(cmd1) <(cmd2)` — compare outputs of two commands
- `cat <(echo hello)` — use command output as file argument
- `tee >(cmd)` — send output to both stdout and a command

## Implementation

### Step 1: Lexer (src/lexer/mod.rs)
Need custom parsing for `<(` and `>(` since they look like redirect + paren.
Add tokens or callback:
```rust
#[regex(r"<\(", parse_process_substitution_in)]
ProcessSubIn(String),  // <(command)

#[regex(r">\(", parse_process_substitution_out)]  
ProcessSubOut(String), // >(command)
```
The callback parses until matching `)`, handling nesting.

Must come BEFORE `StdinRedirect` (`<`) for priority.

### Step 2: AST (src/parser/ast.rs)
Add new Argument variant:
```rust
pub enum Argument {
    // ... existing variants ...
    ProcessSubstitution { command: String, direction: ProcessSubDir },
}

pub enum ProcessSubDir { In, Out }  // <() vs >()
```

### Step 3: Executor
When expanding arguments:
1. Create a named pipe (or use /dev/fd/N)
2. Fork and run the command, connecting its stdout/stdin to the pipe
3. Replace the argument with the pipe path (e.g., `/dev/fd/63`)

On macOS, use `mkfifo` for named pipes or `/dev/fd/N` approach.
