## Task

Add here-string support: `command <<< "string"` passes the string as stdin to the command.

**Current error:** `rush: Expected argument`

## Implementation

### Step 1: Lexer (src/lexer/mod.rs)
Add token for `<<<`. Must come BEFORE `<<` (HereDoc) in token priority.
```rust
#[token("<<<")]
HereString,
```

### Step 2: AST (src/parser/ast.rs)
Add to RedirectKind:
```rust
HereString,  // <<<
```

### Step 3: Parser (src/parser/mod.rs)
In `parse_command` and `parse_single_redirect`, handle the HereString token:
- Consume `<<<`
- Parse the next argument as the string value
- Create `Redirect { kind: RedirectKind::HereString, target: Some(string_value) }`

### Step 4: Executor (src/executor/mod.rs)
When executing a command with HereString redirect:
- Create a pipe
- Write the string + newline to the write end
- Close the write end
- Use the read end as stdin for the command
