## Problem
Every command substitution `$(...)` clones the entire `Runtime`, which includes:
- `history: Option<History>` — up to 10,000 entries with timestamps and a SkimMatcherV2
- `functions: HashMap<String, FunctionDef>` — every user-defined function with full AST bodies
- `variables: HashMap<String, String>` — all shell variables
- `aliases: HashMap<String, String>` — all aliases

This happens in `execute_command_substitution()`, `execute_command_substitution_str()`, and the two `_static` variants in `src/executor/expansion.rs`, plus `execute_subshell()` in `src/executor/commands.rs`.

## Solution
Wrap the heaviest read-mostly fields in `Arc` so cloning is O(1) pointer copy instead of deep copy. History and functions are read-mostly (rarely mutated during command substitutions).

## Steps

### 1. Modify `src/runtime/mod.rs`

Change the `Runtime` struct fields:
```rust
// Before:
history: Option<History>,
functions: HashMap<String, FunctionDef>,

// After:
history: Option<Arc<History>>,
functions: Arc<HashMap<String, FunctionDef>>,
```

Add `use std::sync::Arc;` to imports.

### 2. Update History access methods in Runtime

For `history()` — return `&History`:
```rust
pub fn history(&self) -> &History {
    // Lazy-init still works, just wrap in Arc on first creation
    ...
}
```

For `history_mut()` — use `Arc::make_mut()` for copy-on-write:
```rust
pub fn history_mut(&mut self) -> &mut History {
    let arc = self.history.get_or_insert_with(|| Arc::new(History::new()));
    Arc::make_mut(arc)
}
```

This way, clones share the same History until someone mutates it, at which point only that clone gets a deep copy.

### 3. Update Functions access methods

For reads (get_function, get_all_functions):
```rust
pub fn get_function(&self, name: &str) -> Option<&FunctionDef> {
    self.functions.get(name)
}
```
This should work unchanged since `Arc<HashMap>` derefs to `HashMap`.

For writes (set_function, remove_function):
```rust
pub fn set_function(&mut self, name: String, func: FunctionDef) {
    Arc::make_mut(&mut self.functions).insert(name, func);
}
```

### 4. Update Runtime::new()
```rust
functions: Arc::new(HashMap::new()),
history: None,  // lazy init unchanged
```

### 5. Remove the manual Clone impl for History if one exists
Since History is now behind Arc, the manual Clone impl on History is less important. But keep it — `Arc::make_mut` needs `Clone` on the inner type.

### 6. Fix any compilation errors
- Methods that return `&History` should work since `Arc<T>` implements `Deref<Target=T>`
- Methods that take `&mut History` need `Arc::make_mut()`
- Search for `.history` and `.functions` access across the codebase and fix any issues

### 7. Run full test suite
```bash
cargo test
```

## Files
- `src/runtime/mod.rs` (modify — Arc-wrap fields, update accessors)
- Any files that directly access `runtime.history` or `runtime.functions` fields (fix compile errors)

## Don't
- Don't wrap `variables` or `aliases` in Arc — they're mutated frequently
- Don't change any public API signatures if possible (return &History not Arc<History>)
- Don't add any new dependencies
- Don't change the History struct itself
