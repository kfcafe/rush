## Task
Concatenate adjacent quoted/unquoted strings into a single argument.

## The bug
`echo 'start'"middle"'end'` produces 3 separate arguments with spaces between them
instead of one concatenated argument "startmiddleend".

## POSIX rule
Section 2.2.3: adjacent strings shall be concatenated into a single word.
`'hello'" world"` → one argument: `hello world`

## Steps
1. In src/parser/mod.rs, after parse_argument returns, peek at the next token.
   If the next token is ALSO a string/identifier/variable with NO whitespace between
   them, parse it and concatenate.

   The tricky part: the lexer skips whitespace, so the parser doesn't know if there
   was whitespace between tokens. You may need to:
   - Option A: Check token spans (if available from logos) to see if tokens are adjacent
   - Option B: Handle this at the lexer level by not splitting adjacent quoted segments
   - Option C: In the parser, if the current token is a closing quote and next token
     is an opening quote, combine them

2. The result should be a single Argument. If mixing quoted types, produce
   DoubleQuoted with the parts from each segment.
