## Task

When structured output reaches the end of a pipeline (i.e., will be displayed to the user), auto-render it as a pretty table using the existing `TableRenderer`. When piped to an external command, flatten to text.

## Behavior

1. **Terminal output**: `ls` alone → `Output::Structured` → `TableRenderer` → pretty columns
2. **Piped to external**: `ls | grep foo` → `Output::Structured` → flatten to text lines → pipe to grep
3. **Piped to rush builtin**: `ls | where size > 10mb` → `Output::Structured` stays structured
4. **JSON flag**: `ls --json` → `Output::Structured` → render as JSON text (pretty-printed)
5. **Agent mode**: `Output::Structured` → render as compact JSON

## Approach

The rendering decision happens in the **executor's pipeline handler** and the **main loop's output printing**.

In `run_interactive_with_reedline` (src/main.rs), where it currently does:
```rust
let stdout_text = result.stdout();
if !stdout_text.is_empty() { print!("{}", stdout_text); }
```

Change to:
```rust
match &result.output {
    Output::Structured(value) => {
        // Render as table for terminal, JSON for agent mode
        let rendered = render_structured_output(value, agent_mode);
        if !rendered.is_empty() { print!("{}", rendered); }
    }
    Output::Text(text) => {
        if !text.is_empty() { print!("{}", text); }
    }
}
```

In pipeline execution (src/executor/pipeline.rs), when passing output to external commands:
```rust
// Flatten structured output to text for external commands
let input_bytes = match &result.output {
    Output::Structured(v) => structured_to_text_lines(v).into_bytes(),
    Output::Text(t) => t.clone().into_bytes(),
};
```
