## Problem
`Agent::run()` in `src/ai/agent.rs` pushes every tool call and tool result into `messages: Vec<Message>` without any size limit. Each iteration adds:
- The assistant's tool call message (small)
- The tool result (could be huge — full file contents from `read`, full command output from `shell`)

With 20 iterations, reading several large files, this can grow to hundreds of MB. Worse, the entire history is sent to the LLM on each subsequent request, so network usage compounds too.

## Solution
After adding each tool result, check total conversation size and trim old messages if it exceeds a limit. Keep the system message and the most recent messages.

## Steps
1. Add a helper function in `src/ai/agent.rs`:
   ```rust
   /// Estimate total byte size of the conversation.
   fn estimate_messages_size(messages: &[Message]) -> usize {
       messages.iter().map(|m| m.content.len()).sum()
   }
   
   /// Trim conversation to fit within budget by removing oldest non-system messages.
   /// Keeps: system message (index 0), user intent (index 1), and the most recent messages.
   fn trim_conversation(messages: &mut Vec<Message>, max_bytes: usize) {
       while messages.len() > 4 && estimate_messages_size(messages) > max_bytes {
           // Remove the 3rd message (index 2) — oldest tool interaction
           // Keep [0]=system, [1]=user intent, and the tail
           messages.remove(2);
       }
   }
   ```

2. Add a constant:
   ```rust
   /// Maximum total conversation size in bytes before trimming old messages.
   const MAX_CONTEXT_BYTES: usize = 2 * 1024 * 1024; // 2MB
   ```

3. In the `run()` method, after pushing the tool result message (around line 205), add:
   ```rust
   // Trim old messages to prevent unbounded memory growth
   trim_conversation(&mut messages, MAX_CONTEXT_BYTES);
   ```

4. Also cap individual tool results. After `let result = self.execute_tool(&tool_call, executor)?;` (around line 200), add:
   ```rust
   // Cap individual tool results to prevent a single large file from blowing up context
   let result = if result.len() > 256 * 1024 {
       let mut truncated = result[..256 * 1024].to_string();
       // Ensure UTF-8 boundary
       while !truncated.is_char_boundary(truncated.len()) {
           truncated.pop();
       }
       truncated.push_str("\n[truncated — output exceeded 256KB]");
       truncated
   } else {
       result
   };
   ```

5. Verify it compiles: `cargo check`
