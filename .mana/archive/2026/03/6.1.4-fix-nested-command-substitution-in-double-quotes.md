## Task
Fix nested command substitution inside double-quoted strings.

## The bug
`"outer: $(echo $(echo inner))"` produces `"outer: $(echo  inner )"` instead of
`"outer: inner"`. The inner $(echo inner) is partially expanded but the outer
$(...) is not fully processed.

## Likely cause
The parse_double_quoted_content function (from bean 6.1.3) needs to handle nested
$() correctly by counting parenthesis depth — the same way the lexer parse_command_substitution
does. Look at fn parse_command_substitution in src/lexer/mod.rs for the nesting logic.

## Steps
1. In parse_double_quoted_content, when encountering $( for a CommandSubstitution part,
   track paren depth to find the matching close. Handle nested $() and quoted strings
   inside the substitution.
2. Verify the CommandSubstitution part contains the FULL nested expression.
