## Task
Make single-quoted strings produce Argument::SingleQuoted so the executor knows NOT to
expand variables or command substitutions in them.

## The bug
Currently: Token::SingleQuotedString → Argument::Literal → executor calls
expand_variables_in_literal() → $VAR and $(cmd) get expanded. Wrong!
Single quotes must preserve everything literally.

## Steps
1. In src/parser/mod.rs, change EVERY place that handles Token::SingleQuotedString
   to produce Argument::SingleQuoted instead of Argument::Literal.
   Search: `rg "SingleQuotedString" src/parser/mod.rs`
   Key locations: parse_argument (line ~698), parse_pipe_ask_prompt, parse_word

2. In the executor, add Argument::SingleQuoted match arms to all resolve_argument
   functions. The handling is trivial — just return the string, NO expansion:
   ```rust
   Argument::SingleQuoted(s) => Ok(s.clone()),
   ```
   Files:
   - src/executor/expansion.rs: resolve_argument (line ~394), resolve_argument_static (~876)
   - src/executor/mod.rs: resolve_argument (~2232), resolve_argument_static (~2818)
   - src/executor/pipeline.rs: resolve_argument (~546)
   - src/executor/commands.rs: inline matches (~22)

3. Also update glob expansion guards — SingleQuoted should NOT glob-expand:
   Search: `rg "Argument::Glob" src/executor/` for the glob expansion match arms.
   SingleQuoted should be treated like Literal (no glob expansion).
