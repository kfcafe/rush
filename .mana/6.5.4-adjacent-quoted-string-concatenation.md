Concatenate adjacent quoted strings into a single argument.
`echo 'start'"middle"'end'` should produce one argument "startmiddleend" not three.

The challenge: the lexer skips whitespace (logos skip rule), so by the time the parser
sees tokens, it cannot distinguish 'a'"b" (adjacent, no space) from 'a' "b" (separate args).

Approach: handle this in the LEXER rather than the parser. When the lexer produces a
String or SingleQuotedString token, check if the very next character (no whitespace)
starts another quoted string or identifier. If so, combine them into a single token.

Alternative: modify the logos skip rule to produce optional whitespace tokens, then
check in the parser. Or add span tracking.

Look at how bash handles this: adjacent quote segments are part of the same word.
The key insight is that word splitting happens AFTER quote removal, not during lexing.
