Change parser to emit Argument::DoubleQuoted(Vec<ArgumentPart>) for Token::String
instead of Argument::Literal. This fixes escape sequences like \$ and nested cmd subs.

1. Add fn parse_double_quoted_content(content: &str) -> Vec<ArgumentPart> to Parser.
   Scan the unquoted string char by char:
   - \$ → ArgumentPart::Literal("$")
   - \` → ArgumentPart::Literal("`")
   - \\ → ArgumentPart::Literal("\\")
   - \" → ArgumentPart::Literal("\"")
   - \ before other chars → ArgumentPart::Literal("\\X") (POSIX: keep backslash)
   - $VARNAME → ArgumentPart::Variable("$VARNAME")
   - ${...} → ArgumentPart::BracedVariable("${...}")
   - $((expr)) → ArgumentPart::CommandSubstitution("$((expr))")
   - $(cmd) → ArgumentPart::CommandSubstitution("$(cmd)") — count parens for nesting!
   - `cmd` → ArgumentPart::CommandSubstitution("`cmd`")
   - plain text → ArgumentPart::Literal("text")
   Merge adjacent Literal parts.

2. In parse_argument, change Token::String arm:
   FROM: strip_outer_quotes + process_double_quote_escapes → Argument::Literal
   TO:   strip_outer_quotes → parse_double_quoted_content → Argument::DoubleQuoted

3. Apply same change to other Token::String handlers in parser/mod.rs.
   For contexts that need a plain String (redirects, assignment values, expressions),
   you can still produce a string by expanding parts at parse time for Literal parts
   and keeping $VAR/$() as-is for later expansion. Or wrap in DoubleQuoted where possible.

4. For nested $(cmd), count paren depth like the lexer does (see parse_command_substitution
   in src/lexer/mod.rs for reference). Handle nested quotes inside $().

Fixes: test_double_quotes_escape_dollar, test_double_quotes_escape_backtick,
test_quote_escaping_with_variable, test_nested_command_substitution_with_quotes
