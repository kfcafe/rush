## Task

Add pipeline operators that work on structured data. These are NOT builtins — they are pipeline-native operators that receive `Output::Structured` and produce `Output::Structured`.

## Operators

### `where` — filter rows by condition
```bash
ls | where size > 10mb
ls | where name =~ "*.rs"
git status | where status == "modified"
```

### `sort` — sort by field
```bash
ls | sort size          # ascending
ls | sort -r modified   # descending (reverse)
```

### `select` — pick columns
```bash
ls | select name size
git log | select hash message
```

### `count` — count rows
```bash
ls | count              # → "42"
ls | where size > 1mb | count
```

### `first` / `last` — take N rows
```bash
git log | first 5
ls | sort size | last 3
```

### `uniq` — deduplicate by field
```bash
git log | select author | uniq
```

### Parser changes (src/parser/)
These operators appear in pipeline position. The parser needs to recognize them as special pipeline elements when they follow a `|`.

Add a new variant to `PipelineElement`:
```rust
pub enum PipelineElement {
    Command(Command),
    Subshell(Vec<Statement>),
    CompoundCommand(Box<Statement>),
    StructuredOp(StructuredOp),  // NEW

    Where { field: String, op: CompareOp, value: String },
    Sort { field: String, reverse: bool },
    Select { fields: Vec<String> },

    First(usize),
    Last(usize),
    Uniq { field: Option<String> },
}
```
