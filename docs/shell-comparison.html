TL;DR — Where Rush Fits

Rush is a POSIX-compliant shell written in Rust with 50+ native builtins that run 17-427x faster than GNU coreutils. It targets AI agent workloads and structured-data pipelines while maintaining full script compatibility — something no other modern shell achieves simultaneously.

⚡ Performance

The numbers that matter: how fast can each shell start, how much memory does it consume, and how quickly do common operations execute?

Cold Startup Time

Time to execute shell -c exit with no user config (lower = better)
* Rush daemon mode achieves 0.4ms by keeping a warm process. Benchmarks via hyperfine --shell=none on Debian x86_64.

Memory Footprint

Resident memory of idle interactive session (lower = better)
PowerShell's ~80MB comes from the .NET runtime. Nushell's Rust binary is large but efficient. Rush stays under 10MB with all builtins loaded.

Builtin Command Performance

Time for common operations — Rush's native Rust builtins vs. fork/exec to GNU coreutils (lower = better, log scale)
Fish, Nushell, and PowerShell delegate to external ls/grep/cat (or equivalents), so their times mirror Bash/Zsh plus shell overhead. Rush's builtins use memory-mapped I/O, ripgrep internals, and zero fork/exec — running entirely in-process.

AI Agent Workload Performance

Real-world AI agent tasks — git status loops, find+filter+JSON, complex pipelines (lower = better)
AI agents make hundreds of shell calls per task. Rush averages 8.4x faster than Bash for these workloads. Nushell performs well on structured queries but has higher pipeline overhead. Fish and Zsh track close to Bash since they shell out to the same coreutils.

🎯 Design Philosophy Comparison

Every shell makes a fundamental bet on what matters most. Here's what each one optimizes for.

Positioning Map

X-axis: compatibility (POSIX → novel). Y-axis: performance focus (interactive → programmatic).
Rush occupies a unique position: POSIX-compatible and optimized for programmatic/AI use. No other shell combines both.

Strengths & Trade-offs

What each shell is best at — and what it sacrifices.

Rush
✦ Best at: Builtin speed, POSIX + structured data, AI agent workflows, daemon mode
✧ Trade-off: No plugin ecosystem yet, interactive UX still maturing, macOS/Linux only

Bash
✦ Best at: Universal availability, massive script ecosystem, POSIX standard
✧ Trade-off: No structured data, barebone interactive UX, slow builtins

Zsh
✦ Best at: Plugin ecosystem (Oh My Zsh), Bash compatibility + extensions, customization
✧ Trade-off: Slow with plugins loaded, complex config, no structured data

Fish
✦ Best at: Out-of-box UX, autosuggestions, syntax highlighting, readability
✧ Trade-off: Not POSIX-compliant, can't run Bash scripts, slower startup

Nushell
✦ Best at: Structured data pipelines, type system, data analysis, modern syntax
✧ Trade-off: Not POSIX, slow startup (~30ms+), immature ecosystem, pipeline overhead

PowerShell
✦ Best at: Object pipelines, .NET integration, Windows admin, cross-platform
✧ Trade-off: Very slow startup (~300ms+), 80MB+ memory, alien syntax on Unix
